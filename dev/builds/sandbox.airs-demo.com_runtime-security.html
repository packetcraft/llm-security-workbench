AIRS API Runtime

⚡ Minimal Code Integration: Secure your AI applications with just a few lines of code. The AIRS Python SDK reduces integration complexity by approximately 64%.

Step 1: Installation 

# Install the AIRS Python SDK
pip install pan-aisecurity

Step 2: Initialize SDK

import os
import aisecurity
from aisecurity.scan.inline.scanner import Scanner
from aisecurity.generated_openapi_client.models.ai_profile import AiProfile

# Initialize with your API key (from environment variable)
aisecurity.init(api_key=os.getenv("PANW_AI_SEC_API_KEY"))

# Configure your AI security profile
ai_profile = AiProfile(profile_name="your-security-profile")

# Create scanner instance
scanner = Scanner()

🔒 API Key: Set your API key as an environment variable: export PANW_AI_SEC_API_KEY=your_api_key_here

Step 3: Scan User Prompts (Input Protection) 

from aisecurity.scan.models.content import Content

# Scan user input before processing
user_prompt = "Hello, can you help me with..."

scan_result = scanner.sync_scan(
    ai_profile=ai_profile,
    content=Content(prompt=user_prompt),
    metadata={"app_user": "user_123"}
)

# Check if prompt is safe
if scan_result.action == "block":
    print("⚠️ Malicious prompt detected!")
    # Block the request
else:
    print("✅ Prompt is safe to process")
    # Continue with AI processing

Step 4: Scan AI Responses (Output Protection) 

# After getting response from your AI model
ai_response = "Here's the information you requested..."

# Scan the AI-generated response
output_scan = scanner.sync_scan(
    ai_profile=ai_profile,
    content=Content(
        prompt=user_prompt,
        response=ai_response
    ),
    metadata={"app_user": "user_123"}
)

# Check if response is safe to return
if output_scan.action == "block":
    print("⚠️ Unsafe AI response detected!")
    # Return safe fallback message
    return "I cannot provide that information."
else:
    print("✅ Response is safe to return")
    # Return the AI response to user
    return ai_response

Complete Integration Example 

import os
import aisecurity
from aisecurity.scan.inline.scanner import Scanner
from aisecurity.generated_openapi_client.models.ai_profile import AiProfile
from aisecurity.scan.models.content import Content

# Initialize once at app startup
aisecurity.init(api_key=os.getenv("PANW_AI_SEC_API_KEY"))
scanner = Scanner()
ai_profile = AiProfile(profile_name="your-security-profile")

def process_user_query(user_id, user_prompt):
    """
    Secure AI query processing with dual-phase scanning
    """
    # PHASE 1: Scan user input
    input_scan = scanner.sync_scan(
        ai_profile=ai_profile,
        content=Content(prompt=user_prompt),
        metadata={"app_user": user_id}
    )

    if input_scan.action == "block":
        return "Your request violates security policies."

    # Process with your AI model (OpenAI, Anthropic, etc.)
    ai_response = your_ai_model(user_prompt)

    # PHASE 2: Scan AI response
    output_scan = scanner.sync_scan(
        ai_profile=ai_profile,
        content=Content(prompt=user_prompt, response=ai_response),
        metadata={"app_user": user_id}
    )

    if output_scan.action == "block":
        return "I cannot provide that information."

    return ai_response

# Example usage
result = process_user_query("user_123", "Tell me about your products")

AIRS Network Intercept Onboarding

🌐 Zero-Code Network Integration: Deploy AI security at the network layer without modifying application code. Protect all AI traffic flowing through your infrastructure.

Onboarding Overview

Network Intercept provides transparent AI security by intercepting and inspecting AI API traffic at the network level. Follow these steps to onboard your cloud account:

Step 1: Access Strata Cloud Manager

Navigate to: Strata Cloud Manager → AI Runtime Security → Settings → Cloud Accounts

This is where you'll configure your cloud infrastructure to work with AIRS Network Intercept.

Step 2: Onboard Cloud Account

  1. Click "Onboard Cloud Account"
  2. Select your cloud provider:
    • AWS
    • Azure
    • Google Cloud Platform
  3. Enter your cloud account credentials
  4. Configure network access permissions

Step 3: AWS Integration (Example) 

# AWS CloudFormation Template Configuration
# Deploy this stack to enable AIRS Network Intercept

Resources:
  AIRSRole:
    Type: AWS::IAM::Role
    Properties:
      RoleName: PrismaAIRS-NetworkIntercept
      AssumeRolePolicyDocument:
        Version: '2012-10-17'
        Statement:
          - Effect: Allow
            Principal:
              AWS: arn:aws:iam::PANW_ACCOUNT:root
            Action: sts:AssumeRole
      ManagedPolicyArns:
        - arn:aws:iam::aws:policy/ReadOnlyAccess
      Policies:
        - PolicyName: AIRSNetworkAccess
          PolicyDocument:
            Version: '2012-10-17'
            Statement:
              - Effect: Allow
                Action:
                  - ec2:DescribeInstances
                  - ec2:DescribeNetworkInterfaces
                  - ec2:DescribeVpcs
                  - elasticloadbalancing:DescribeLoadBalancers
                Resource: '*'

🔐 Security Note: Replace PANW_ACCOUNT with the Palo Alto Networks account ID provided in Strata Cloud Manager.

Step 4: Configure Network Intercept Points

Specify which traffic should be inspected:

  1. API Gateway Integration: Intercept at load balancer or API gateway level
  2. VPC Traffic Mirroring: Mirror relevant AI API traffic for inspection
  3. Subnet Configuration: Select subnets containing AI workloads
  4. Traffic Filtering: Define rules for AI-related traffic patterns

Step 5: Activate Security Profiles

Create and apply AI Security Profiles:

  1. Navigate to AI Runtime Security → AI Profiles
  2. Create a new profile or select existing
  3. Configure protection policies:
    • Prompt injection detection
    • Malicious payload filtering
    • Data loss prevention
    • Toxic content blocking
  4. Apply profile to network intercept points

Step 6: Verify Deployment 

# Test Network Intercept with curl

curl -X POST https://your-ai-api.example.com/chat \
  -H "Content-Type: application/json" \
  -d '{

# Check Strata Cloud Manager logs to confirm:
# 1. Traffic is being intercepted
# 2. Security policies are being applied
# 3. Threats are being detected and blocked

Step 7: Monitor and Optimize

Access real-time monitoring dashboards:

  1. Traffic Analytics: View all intercepted AI API calls
  2. Threat Detection: Monitor blocked malicious requests
  3. Performance Metrics: Track latency and throughput
  4. Compliance Reports: Generate audit trails for AI usage

🎯 Key Benefits: Zero code changes, comprehensive coverage, centralized policy management, detailed visibility into all AI traffic, and automatic protection for new AI services.

⚠️ Important: Network Intercept requires proper network architecture planning. Work with your network team to ensure minimal latency impact and proper traffic routing.
